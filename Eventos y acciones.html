Lección 1
Eventos y acciones

En la programación por bloques, los programas no comienzan solos: necesitan que algo ocurra para ponerse en marcha. Ese “algo” se conoce como evento.

Cuando un evento sucede, el programa responde ejecutando una o varias acciones. Esta relación entre evento y acción es la base del funcionamiento de muchos programas interactivos como juegos, simulaciones y aplicaciones educativas.

¿Qué es un evento?

Un evento ocurre cuando sucede algo específico, por ejemplo: hacer clic en la bandera verde, presionar una tecla o tocar la pantalla.

Un evento es una situación que activa el inicio de un programa o de una parte del mismo. En programación por bloques, los eventos suelen representarse con bloques que indican “cuando ocurra algo…”.

Algunos eventos comunes son:

  • Cuando se hace clic en la bandera verde.
  • Cuando se presiona una tecla.
  • Cuando se hace clic sobre un objeto.
  • Cuando se recibe un mensaje.

¿Qué es una acción?

Las acciones son las tareas que realiza el programa después de que ocurre un evento.

Una acción es una instrucción que el programa ejecuta como respuesta a un evento. Las acciones indican qué debe hacer el personaje, objeto o sistema.

Ejemplos de acciones en programación por bloques:

  • Mover un personaje.
  • Reproducir un sonido.
  • Mostrar un mensaje.
  • Cambiar de color o de escenario.

Ejemplo 1

Relacione el evento con la acción correspondiente en un programa por bloques.

Objetivo: Comprender la relación evento → acción.

Inicio

  1. Ocurre el evento: se presiona la bandera verde.
  2. El programa inicia.
  3. El personaje aparece en pantalla.
  4. El personaje se mueve hacia adelante.

Fin

En este ejemplo, la bandera verde es el evento y mover el personaje es la acción.

Ejemplo 2

Analice la siguiente situación:

Objetivo: Crear una interacción sencilla con el teclado.

Inicio

  1. Evento: se presiona la tecla espacio.
  2. Acción: el personaje salta.
  3. Acción: se reproduce un sonido.

Fin
Presiona la tecla ESPACIO para activar el evento.

Observe que un mismo evento puede activar varias acciones al mismo tiempo. Esto permite crear programas más dinámicos e interactivos.

En programación por bloques, los eventos funcionan como disparadores y las acciones como respuestas. Esta lógica está presente en videojuegos, robots, simulaciones y sistemas interactivos. Comprender esta relación facilita el diseño de programas claros, organizados y fáciles de entender.
