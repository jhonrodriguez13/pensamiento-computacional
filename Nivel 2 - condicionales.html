Lección 1
Condicionales en programación por bloques

En programación por bloques, las decisiones son fundamentales. Muchas veces no basta con ejecutar acciones de forma lineal: el programa necesita evaluar condiciones para decidir si hacer algo o no.

Los condicionales permiten que un programa tome decisiones en función de si una condición es verdadera o falsa. Esta habilidad es clave para que los programas sean más inteligentes, dinámicos y capaces de adaptarse a lo que ocurre durante su ejecución.

¿Qué es un condicional?

Un condicional típico tiene la forma si (condición) entonces (acción).

En entornos de programación por bloques como Scratch, blockly, Code.org los condicionales se representan con bloques que permiten encajar dentro de ellos otras instrucciones. El programa las ejecuta solo si la condición evaluada es verdadera.

Por ejemplo, un condicional puede evaluar si un personaje está tocando un color, si se ha presionado una tecla o si un valor es mayor o menor que otro. Si la condición es verdadera, el programa ejecuta la acción; si no lo es, el programa continúa sin cambiar.

Condicional: si – entonces – si no

En muchos programas no basta con ejecutar una acción solo cuando una condición se cumple. También es importante definir qué debe hacer el programa cuando esa condición no se cumple. Para eso se utiliza el condicional si – entonces – si no.

Este tipo de condicional permite que el programa tome dos caminos diferentes, dependiendo del resultado de una condición. De esta forma, el programa puede reaccionar de manera más inteligente y adaptarse a distintas situaciones.

El bloque si – entonces – si no permite ejecutar una acción cuando la condición es verdadera y otra diferente cuando es falsa.

El funcionamiento del condicional si – entonces – si no es el siguiente:

  • Primero, el programa evalúa una condición.
  • Si la condición es verdadera, ejecuta las acciones del bloque entonces.
  • Si la condición es falsa, ejecuta las acciones del bloque si no.

Por ejemplo, en un videojuego educativo:

Inicio

  1. Condición: ¿el personaje tocó la meta?
  2. Si es verdadero → mostrar el mensaje “¡Ganaste!”
  3. Si no → mostrar el mensaje “Ánimo ya casi llegas”

Fin
Pulsa la imagen para ver el bloque condicional si entonces sino.

Gracias a este tipo de condicional, los programas pueden ofrecer retroalimentación, tomar decisiones automáticas y crear experiencias interactivas más completas, como juegos, simulaciones y actividades educativas.

Ejemplo con “si – entonces – si no”

A menudo necesitamos que el programa tome una acción si la condición es verdadera, y otra distinta si es falsa. Este tipo de condicional se llama si – entonces – si no.

Inicio

  1. Condición: ¿el puntaje es mayor o igual a 10?
  2. Si es verdadero → mostrar “¡Nivel completado!”
  3. Si es falso → mostrar “Sigue intentando”

Fin

Esta estructura permite que un programa tenga múltiples respuestas según los resultados de una condición.

Actividad 1 - Explorar condiciones

Instrucciones: Usa Scratch o tu editor por bloques favorito y crea un proyecto donde:

  1. El personaje camine hacia adelante si toca un color específico.
  2. El personaje diga “¡Hola!” si se presiona la tecla espacio.
  3. El fondo cambie si el puntaje es mayor que 5, de lo contrario no cambie.

Piensa en las condiciones que estás evaluando y responde:

  • ¿Qué sucede si ninguna condición se cumple?
  • ¿Puedes encadenar varios condicionales para lograr comportamientos más complejos?

Actividad 2 – Juego de decisiones

Crea un pequeño juego por bloques (como un laberinto o mini juego) donde:

  • Si el personaje recoge una llave → abre una puerta.
  • Si el personaje toca un obstáculo → pierde una vida.
  • Si el tiempo se acaba → muestra “Intenta de nuevo”.

Estas decisiones lógicas deben ocurrir usando bloques condicionales como “si … entonces” y “si … entonces … si no”.

Reflexión

Los condicionales permiten que los programas respondan a distintos estímulos y puedan tomar decisiones inteligentes. En la vida real también usamos condicionales constantemente:

  • Si hace frío → me pongo un abrigo, si no → uso camiseta.
  • Si mi puntaje es alto → recibo premio, si no → sigo practicando.

Entender cómo funcionan las condiciones en programación te ayuda a diseñar mejores soluciones lógicas y creativas para tus proyectos.
